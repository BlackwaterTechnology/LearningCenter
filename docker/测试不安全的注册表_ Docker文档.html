测试不安全的注册表

预计阅读时间： 4分钟

强烈建议使用已知CA颁发的TLS证书保护注册表，您可以选择使用自签名证书，也可以通过未加密的HTTP连接使用注册表。这些选择之一涉及安全性权衡和其他配置步骤。

部署纯HTTP注册表

警告：不可能使用不安全的注册表进行基本身份验证。

此过程配置Docker完全忽略注册表的安全性。这是非常不安全的，不推荐。它将您的注册表暴露给微不足道的中间人（MITM）攻击。只能使用此解决方案进行隔离测试，或在严格控制，空气密集的环境中使用。

  1. 编辑daemon.json文件，其默认位置 /etc/docker/daemon.json在Linux或 C:\ProgramData\docker\config\daemon.jsonWindows Server上。如果您使用Docker for Mac或Docker for Windows，请单击Docker图标，选择 “首选项”，然后选择“+ 守护进程”。

    如果daemon.json文件不存在，请创建它。假设文件中没有其他设置，它应该具有以下内容：

    {
  "insecure-registries" : ["myregistrydomain.com:5000"]
}

    将示例中的不安全注册表的地址替换为该安全注册表的地址。

    通过启用不安全的注册表，Docker将执行以下步骤：

    • 首先，尝试使用HTTPS。
      • 如果HTTPS可用但证书无效，请忽略有关证书的错误。
      • 如果HTTPS不可用，请返回HTTP。

  2. 重新启动Docker以使更改生效。

在要访问注册表的每个引擎主机上重复这些步骤。

使用自签名证书

警告：与基本身份验证一起使用也需要将证书信任到某些版本的Docker的OS证书存储区（见下文）

这比不安全的注册表解决方案更安全。

  1. 生成自己的证书：

    $ mkdir -p certs

$ openssl req \
  -newkey rsa:4096 -nodes -sha256 -keyout certs/domain.key \
  -x509 -days 365 -out certs/domain.crt

    确保使用该名称myregistrydomain.com作为CN。

  2. 使用结果启动您的注册表启用TLS。

  3. 指示每个Docker守护进程信任该证书。这样做取决于您的操作系统。

    • Linux：将domain.crt文件复制到 /etc/docker/certs.d/myregistrydomain.com:5000/ca.crt每个Docker主机上。您不需要重新启动Docker。

    • Windows Server：

      1. 打开Windows资源管理器，右键单击该domain.crt 文件，然后选择安装证书。出现提示时，选择以下选项：

        商店位置 本地机器
        将所有证书放在以下商店中 选

      2. 单击浏览器并选择受信任的根证书颁发机构。

      3. 单击完成。重启Docker。

    • Docker for Mac：按照添加自定义CA证书的说明进行操作 。重启Docker。

    • Docker for Windows：按照添加自定义CA证书的说明进行操作 。重启Docker。

排除不安全的注册表

本节列出了一些常见的故障以及如何从中恢复。

失败…

如果配置引擎守护进程并尝试从不使用TLS的注册表中引用，则会导致以下消息：

FATA[0000] Error response from daemon: v1 ping attempt failed with error:
Get https://myregistrydomain.com:5000/v1/_ping: tls: oversized record received with length 20527.
If this private registry supports only HTTP or HTTPS with an unknown CA certificate,please add
`--insecure-registry myregistrydomain.com:5000` to the daemon's arguments.
In the case of HTTPS, if you have access to the registry's CA certificate, no need for the flag;
simply place the CA certificate at /etc/docker/certs.d/myregistrydomain.com:5000/ca.crt

Docker在使用身份验证时仍然抱怨该证书？

使用身份验证时，Docker的某些版本还要求您在操作系统级别信任证书。

Ubuntu的

$ cp certs/domain.crt /usr/local/share/ca-certificates/myregistrydomain.com.crt
update-ca-certificates

红帽企业版Linux

cp certs/domain.crt /etc/pki/ca-trust/source/anchors/myregistrydomain.com.crt
update-ca-trust

Oracle Linux

$ update-ca-trust enable

重新启动Docker以使更改生效。

视窗

打开Windows资源管理器，右键单击证书，然后选择 安装证书。

然后，选择以下选项：

  • 店铺位置：本地机
  • 检查所有证书在以下商店
  • 单击浏览器，然后选择受信任的根证书颁发机构
  • 单击完成

进一步了解如何管理TLS证书。

将CA证书添加到Windows后，重新启动Docker for Windows。
